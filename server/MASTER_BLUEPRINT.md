# 🗺️ المخطط الشامل للمشروع (Master Project Blueprint)
## منصة الحرف اليدوية والطباعة الرقمية (Handcraft & Digital Print Platform)

هذا الملف هو "الدستور التقني" للمشروع، ويجمع بين الرؤية الفنية والمتطلبات التنفيذية لكل فريق (باك اند، فرونت اند، ديف أوبس).

---

## 🚀 1. الرؤية والهدف (The Vision)
إنشاء سوق إلكتروني متكامل يربط الحرفيين (Artisans) بالمشترين، مع ميزة فريدة وهي بيع **التصاميم الرقمية (Digital Patterns)** بجانب المنتجات اليدوية، وتوفير نظام تفاوض حي (Negotiation Engine) للطلبات الخاصة.

---

## 👥 2. أدوار المستخدمين (User Roles)
1.  **المشتري (Buyer):** تصفح، شراء، تتبع طلبات، طلب تصاميم خاصة، تحميل ملفات رقمية.
2.  **الحرفي (Artisan):** إدارة متجر، رفع منتجات/تصاميم، تحديد أسعار، الرد على طلبات التفاوض، طلب سحب الأرباح.
3.  **المدير (Admin):** رقابة شاملة، اعتماد الحرفيين، إدارة العمولات، معالجة الدفعات المالية.

---

## 🛠️ 3. التوصيات التقنية (Tech Stack)

### **الباك اند (Backend) - "العقل المدبر"**
*   **اللغة:** Node.js (TypeScript) مع Express.
*   **قاعدة البيانات:** MongoDB (Mongoose) للبيانات المرنة، و Redis للتخزين المؤقت (Caching) والـ Real-time.
*   **الملفات:**
    *   **Cloudinary:** للصور العامة (صور المنتجات، الأفاتار).
    *   **AWS S3 Private:** حصرياً للملفات الرقمية (SVG, PDF) لضمان عدم وصول أي شخص لها إلا بعد الدفع.
*   **الأمان:** JWT (Access/Refresh Tokens) + HttpOnly Cookies.

### **الفرونت اند (Frontend) - "الواجهة الجذابة"**
*   **الإطار:** Next.js (App Router) لضمان أفضل SEO وسرعة أداء.
*   **التصميم:** Tailwind CSS + Shadcn UI (للحصول على واجهة عصرية واحترافية).
*   **إدارة الحالة:** TanStack Query (React Query) للتعامل مع البيانات، و Zustand للحالة العامة.
*   **الأدوات:** Fabric.js للسماح للمستخدمين بتعديل ألوان التصاميم الرقمية ومعاينتها قبل الشراء.

## 👷 5. إرشادات الفرق (Team Guidance)

### **إلى مطور الباك اند (Backend Guide):**
1.  **الأمان المالي:** لا تقم أبداً بتحديث رصيد الحرفي بناءً على طلب من الفرونت اند؛ استخدم **Webhooks** من Stripe/Paymob دائماً.
2.  **معالجة الصور:** استخدم مكتبة `Sharp` لضغط الصور وتصغير حجمها قبل رفعها لـ Cloudinary لتوفير المساحة وسرعة التحميل.
3.  **الملفات الرقمية:** تأكد أن ملفات الـ S3 ليست "Public". الوصول إليها يتم فقط عبر توليد `Signed URL` تنتهي صلاحيته بعد 5 دقائق.

### **إلى مطور الفرونت اند (Frontend Guide):**
1.  **تجربة المستخدم (UX):** المنصة يجب أن تكون **Multi-language** (AR/EN). استخدم `next-intl` أو `i18next`.
2.  **الأداء:** استخدم صور `next/image` لضمان التحميل الكسول (Lazy Loading) والتحسين التلقائي.
3.  **المعاينات التفاعلية:** في صفحة التصاميم الرقمية، استخدم `Fabric.js` لعمل محاكي بسيط يغير ألوان التصميم ليراها المستخدم قبل الشراء.

### **إلى مهندس الديف أوبس (DevOps & Deploy Guide):**
1.  **الاستضافة:**
    *   الباك اند: **Render** أو **Railway**.
    *   الفرونت اند: **Vercel** أو **Netlify**.
2.  **CI/CD:** إعداد GitHub Actions لعمل Build واختبار تلقائي عند كل Push.
3.  **المراقبة:** استخدام **Sentry** لتتبع الأخطاء البرمجية في الإنتاج فور حدوثها.
